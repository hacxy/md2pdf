# md2pdf — 产品需求文档（PRD）

## 概述

md2pdf 是一款基于浏览器的 Markdown 转 PDF 在线工具。用户可以直接在浏览器中粘贴或上传 Markdown 文档，实时预览渲染效果，选择不同的内置主题，一键导出为精美排版的 PDF 文件。整个过程无需注册登录、无需安装任何软件，打开网页即用。

## 背景与问题陈述

### 当前痛点

- **Typora 等桌面工具**：需要安装软件，且 Typora 已转为付费（$14.99），主题定制门槛较高
- **在线 Markdown 编辑器**（如 StackEdit、Dillinger）：侧重编辑而非导出，PDF 导出功能弱，主题选择少
- **VS Code + 插件**：需要技术背景，对通用用户不友好
- **手动排版**：将 Markdown 内容复制到 Word/Google Docs 再导出 PDF，流程繁琐且格式容易丢失

### 如果不做这个产品

用户只能在"付费桌面软件"、"功能简陋的在线工具"、"需要技术背景的开发者工具"之间做取舍，没有一个免费、好用、开箱即用的浏览器端 Markdown → PDF 方案。

## 目标与非目标

### 目标

- **G1**: 用户在 30 秒内完成"打开网页 → 粘贴 Markdown → 选择主题 → 导出 PDF"全流程
- **G2**: 提供至少 5 个精心设计的内置主题，覆盖技术文档、学术报告、简约笔记等常见场景
- **G3**: 正确渲染 GFM 标准 Markdown 语法 + 代码块语法高亮
- **G4**: 导出的 PDF 排版质量不低于 Typora 的默认主题

### 非目标

- 不做 Markdown 编辑器（不需要完整的编辑功能如快捷键、自动补全等）
- 不做用户系统（无注册、登录、历史记录）
- 不做云端存储（文件不上传服务器）
- 不做协作功能
- 不支持 Markdown 扩展语法（如数学公式、Mermaid 图表），可作为后续版本迭代

## 用户体验设计

### 用户旅程

```
打开网页 → 输入/上传 Markdown → 实时预览 → 选择主题 → 导出 PDF
```

**1. 入口（Entry）**

用户通过浏览器直接访问 md2pdf 网站，首页即是工具界面，无需任何前置操作。页面首次加载时展示一段示例 Markdown 内容，帮助用户理解工具用途。

**2. 核心交互（Core Interaction）**

- **左侧**：Markdown 输入区域（文本框 / 文件上传）
- **右侧**：实时渲染预览区域
- **顶部工具栏**：主题选择器、导出按钮
- 用户可以：
  - 直接在文本框中粘贴/输入 Markdown 内容
  - 通过"上传文件"按钮选择本地 `.md` 文件
  - 拖拽 `.md` 文件到页面上传
  - 在主题选择器中切换主题，预览区实时更新
  - 点击"导出 PDF"一键下载

**3. 完成（Completion）**

- PDF 文件自动下载到本地，文件名默认取 Markdown 第一个 `# 标题`，无标题时使用 `document.pdf`
- 下载完成后页面显示短暂的成功提示（toast）
- 用户可继续修改内容或切换主题后再次导出

### 摩擦与愉悦

**潜在摩擦点：**

| 摩擦点 | 缓解方案 |
|--------|----------|
| 用户不知道什么是 Markdown | 提供默认示例内容，附带简要说明 |
| 大文件渲染慢 | 使用防抖渲染，展示加载状态 |
| 导出的 PDF 分页不理想 | 精心设计 print CSS，避免标题和内容分离 |
| 不确定哪个主题合适 | 主题切换时预览区实时更新，快速对比 |

**愉悦设计：**

- 主题切换动画平滑，视觉上有"变装"的愉悦感
- 默认示例内容展示各种 Markdown 语法的渲染效果，用户一打开就能感受到产品质量
- 导出 PDF 的速度极快（浏览器原生能力），给人"即点即得"的感觉

### 边缘情况与错误处理

| 场景 | 处理方式 |
|------|----------|
| 输入为空时点击导出 | 禁用导出按钮，或提示"请先输入 Markdown 内容" |
| 上传非 `.md` 文件 | 提示"请上传 .md 或 .txt 文件" |
| 超大文件（>1MB） | 提示文件过大，建议分割 |
| 浏览器不支持所需 API | 显示兼容性提示，推荐使用 Chrome/Edge |

## 功能需求

### FR1: Markdown 输入

- **FR1.1**: 提供文本输入区域，支持直接粘贴或键入 Markdown 内容
- **FR1.2**: 支持通过文件选择器上传 `.md` 或 `.txt` 文件
- **FR1.3**: 支持拖拽文件到页面进行上传
- **FR1.4**: 输入区域支持自动缩进和基础的文本编辑体验
- **FR1.5**: 首次访问时展示默认示例 Markdown 内容

### FR2: Markdown 渲染

- **FR2.1**: 支持完整的 GFM（GitHub Flavored Markdown）语法
  - 标题（h1-h6）、段落、换行
  - 粗体、斜体、删除线
  - 有序列表、无序列表、任务列表
  - 表格
  - 链接、图片（支持外部 URL）
  - 引用块
  - 分割线
- **FR2.2**: 代码块支持语法高亮，覆盖主流编程语言（JavaScript、TypeScript、Python、Go、Java、C/C++、Rust、HTML、CSS、SQL、Shell/Bash、JSON、YAML 等）
- **FR2.3**: 行内代码使用等宽字体和背景色区分
- **FR2.4**: 渲染结果实时更新，输入变化后 300ms 内响应（防抖处理）

### FR3: 主题系统

- **FR3.1**: 提供至少 5 个内置主题：
  - **GitHub**：模拟 GitHub 的 Markdown 渲染风格，适合技术文档
  - **学术（Academic）**：类 LaTeX 风格，衬线字体，适合论文/报告
  - **简约（Minimal）**：大量留白，无衬线字体，现代简洁风
  - **商务（Business）**：专业配色，适合正式商务文档
  - **暗夜（Dark）**：深色背景主题，适合技术分享和演示
- **FR3.2**: 主题切换时预览区实时更新，无需重新渲染 Markdown
- **FR3.3**: 每个主题包含完整的排版规则：字体、字号、行高、间距、颜色、代码高亮配色

### FR4: PDF 导出

- **FR4.1**: 点击导出按钮生成 PDF 并自动下载
- **FR4.2**: PDF 文件名规则：取文档第一个 `#` 标题作为文件名；无标题时使用 `document.pdf`
- **FR4.3**: PDF 分页合理，避免以下情况：
  - 标题出现在页面最底部（孤标题）
  - 代码块被截断跨页
  - 表格在不合适的位置断开
- **FR4.4**: PDF 中的链接可点击（保留超链接）
- **FR4.5**: 支持选择页面尺寸：A4（默认）、Letter、A3
- **FR4.6**: 导出过程展示加载状态

### FR5: 界面交互

- **FR5.1**: 响应式布局，在桌面端左右分栏（输入 + 预览），移动端上下分栏或可切换标签
- **FR5.2**: 分栏比例可拖拽调整
- **FR5.3**: 支持纯预览模式（隐藏输入区，全屏预览）
- **FR5.4**: 提供深色/浅色界面模式切换（独立于文档主题）
